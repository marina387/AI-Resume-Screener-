RAG (Retrieval-Augmented Generation) — подход, при котором модель сначала находит релевантные фрагменты документов,
а затем генерирует ответ на основе найденного контекста. Ключевые шаги:
1) Загрузка документов
2) Разбиение на чанки
3) Векторизация (эмбеддинги)
4) Поиск похожих чанков (retrieval)
5) Ответ на основе контекста (в нашем MVP — просто вывод контекста + шаблон ответа)
